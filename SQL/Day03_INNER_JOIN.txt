# 📌 학습 목표

이번 학습에서는 두 개 이상의 테이블을 연결하는 JOIN의 개념과
가장 많이 사용하는 INNER JOIN을 학습한다.

학습 내용

- JOIN이 필요한 이유
- INNER JOIN
- ON
- Primary Key(PK)
- Foreign Key(FK)

---

# 📚 핵심 문법

## 1️⃣ INNER JOIN

```sql
SELECT *
FROM employees
JOIN departments
ON employees.department_id = departments.department_id;
```

✔ 두 테이블에서 공통된 데이터만 조회한다.

---

## 2️⃣ ON

```sql
ON employees.department_id = departments.department_id;
```

✔ 두 테이블을 어떤 컬럼으로 연결할지 지정한다.

보통 Primary Key(PK)와 Foreign Key(FK)를 연결한다.

---

## 3️⃣ Alias와 함께 사용

```sql
SELECT e.name,
       d.department_name
FROM employees e
JOIN departments d
ON e.department_id = d.department_id;
```

✔ Alias를 사용하면 SQL이 훨씬 읽기 쉬워진다.

---

# 💡 핵심 개념

| 문법 | 설명 |
|------|------|
| JOIN | 테이블 연결 |
| INNER JOIN | 공통 데이터 조회 |
| ON | 연결 조건 |
| PK | 고유한 값 |
| FK | 다른 테이블을 참조하는 값 |
| Alias | 테이블 이름을 줄여 작성 |

---

# 📝 배운 점

- JOIN은 여러 테이블의 데이터를 함께 조회하기 위해 사용한다.
- INNER JOIN은 공통된 데이터만 조회한다.
- ON 절을 사용하여 연결 기준을 지정한다.
- 실무에서는 Alias와 함께 사용하는 경우가 많다.

---

# 🎯 실무 Tip

실무에서는 SELECT *보다 필요한 컬럼만 조회하는 것이 좋다.

```sql
SELECT e.name,
       d.department_name
FROM employees e
JOIN departments d
ON e.department_id = d.department_id;
```

✔ 필요한 데이터만 조회하면 성능과 가독성이 좋아진다.

---

# 💼 금융권 활용 예시

은행에서는 고객 정보와 계좌 정보가 서로 다른 테이블에 저장되는 경우가 많다.

고객 이름과 계좌번호를 함께 조회할 때 JOIN을 사용한다.

```sql
SELECT c.customer_name,
       a.account_number
FROM customers c
JOIN accounts a
ON c.customer_id = a.customer_id;
```

✔ 금융권 데이터 분석에서 JOIN은 가장 자주 사용하는 문법 중 하나이다.
