# 📚 왜 JOIN이 필요할까?

실무에서는 하나의 테이블에 모든 데이터를 저장하지 않는다.

예를 들어,

**employees**

| employee_id | name | department_id |
|-------------|------|---------------|
| 1 | Kim | 10 |
| 2 | Lee | 20 |

그리고

**departments**

| department_id | department_name |
|---------------|-----------------|
| 10 | Finance |
| 20 | HR |

직원의 이름과 부서명을 함께 조회하려면 두 테이블을 연결해야 한다.

이때 사용하는 것이 **JOIN**이다.

---

# 📚 INNER JOIN

```sql
SELECT *
FROM employees
JOIN departments
ON employees.department_id = departments.department_id;
```

✔ 두 테이블에서 공통된 값만 조회한다.

---

# 📚 ON

```sql
ON employees.department_id = departments.department_id
```

ON은 두 테이블을 어떤 컬럼으로 연결할지 지정하는 역할을 한다.

보통 Primary Key와 Foreign Key를 기준으로 연결한다.

---

# 💡 Primary Key(PK)

Primary Key는 각 행을 유일하게 구분하는 값이다.

예시

| employee_id | name |
|--------------|------|
| 1 | Kim |
| 2 | Lee |

employee_id는 중복될 수 없다.

---

# 💡 Foreign Key(FK)

Foreign Key는 다른 테이블의 Primary Key를 참조하는 컬럼이다.

예시

| department_id |
|---------------|
| 10 |
| 20 |

employees.department_id

↓

departments.department_id

를 참조한다.

---

# 💡 핵심 개념

| 문법 | 설명 |
|------|------|
| JOIN | 테이블 연결 |
| INNER JOIN | 공통 데이터만 조회 |
| ON | 연결 조건 |
| PK | 고유한 값 |
| FK | 다른 테이블을 참조하는 값 |

---

# 📝 배운 점

- JOIN은 여러 테이블의 데이터를 함께 조회하기 위해 사용한다.
- INNER JOIN은 공통으로 존재하는 데이터만 조회한다.
- ON 절을 사용하여 어떤 컬럼을 기준으로 연결할지 지정한다.
- 대부분 Primary Key와 Foreign Key를 이용해 JOIN을 수행한다.

---

# 🎯 실무 Tip

실무에서는 `SELECT *`보다 필요한 컬럼만 조회하는 것이 좋다.

```sql
SELECT e.name,
       d.department_name
FROM employees e
JOIN departments d
ON e.department_id = d.department_id;
```

이처럼 Alias를 함께 사용하면 SQL이 더 간결하고 읽기 쉬워진다.

---

# 💼 금융권 활용 예시

은행에서는 고객 정보와 계좌 정보를 각각 다른 테이블에 저장하는 경우가 많다.

예를 들어 고객 이름과 계좌번호를 함께 조회하려면 JOIN을 사용한다.

```sql
SELECT c.customer_name,
       a.account_number
FROM customers c
JOIN accounts a
ON c.customer_id = a.customer_id;
```

JOIN은 금융권 데이터 분석에서도 가장 자주 사용하는 SQL 문법 중 하나이다.
